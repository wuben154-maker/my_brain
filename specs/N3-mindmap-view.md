# N3 — 思维导图 · 分层大纲（`mindmap-view`）

- **阶段：** B/C（建议 B 段后期或 C 段，非 MVP 必做）· **状态：** ✅ 已实现
- **上游：** N0、现有图谱与边· **下游：** 无

## 1. 目标
把「思维导图」导航项做成图谱的**分层大纲/树视角**：以 BFS 从高度数节点展开成可折叠的层级列表（对应 PRODUCT.md「可缩放分层：看脑区团 → 点进去展开细节」），作为力导星图之外的结构化阅读方式，缓解节点膨胀时的可读性。

## 2. 非目标
- 不是第二套图谱编辑器：增删改仍走星图 + 收件箱（不变量 2/6）。
- 不做自由拖拽重排（只读/可折叠展开）。

## 3. 契约
```
src/components/mindmap/MindmapOutline.tsx   // 可折叠层级树（节点标题+简介摘要+定位星图）
src/lib/graphOutline.ts                     // 纯函数：图谱→分层树（复用/扩展 GraphStatsCard 的 BFS）
```
- 数据：`useGraphStore` 可见节点/边；`graphOutline` 输出 `{ node, depth, children }` 树（环用 visited 去重）。
- 动作：点击条目「在星图中查看」→ 切 graph 分区 + 选中高亮（同 N2）。
- 挂载：`NAV_SECTIONS` 中 `mindmap` 改 `live`，分区渲染 `<MindmapOutline>`。
- 空/单节点态有合理展示。

## 4. 验收清单
- [x] 树结构与图谱连通性一致；有环不死循环、不重复展开。
- [x] 折叠/展开正常；定位回链星图选中正确节点。
- [x] 归档节点默认隐藏。
- [x] 截图留证。

## 5. 测试（harness）
- `graphOutline.test.ts`：分层正确、环安全、归档过滤、空集/单点。
- `MindmapOutline` 交互测试：折叠展开 + 定位动作。
