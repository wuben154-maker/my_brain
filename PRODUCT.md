# my_brain — 产品设计文档 v2（沉浸式语音伴侣）

> **v2 大换血**：从「多分区工具仪表盘 + 逐条审批」转向「**一个会说话的伴侣 + 一张自己长大的星图**」。其余繁杂、需手操作的 UI 全部砍掉。
> v1 文档见 git 历史。配套 `AGENTS.md` 给 AI 编码助手作技术约束；落地工作单见 `specs/V*.md`（V 系列）；旧 A/B/C/N/G spec 标 `superseded`，其底层逻辑择优复用。

## 一句话定义
一个**沉浸式、语音优先的 AI 伴侣**：从开机起就用语音陪你，像真人一样能唠家常、开玩笑、被你随时打断；核心是**用人话讲最新 AI 资讯**，你一句话决定是否入库；入库后它**自动**把知识归纳进一张会自我净化的"大脑星图"。用得越久，越懂你。整个应用只有两件事：**和它聊天** + **看着自己的脑图长大**。

---

## 〇、v2 三条产品决策（凌驾旧设计）
1. **入库 = 你决定；整理 = 它自动**。是否把一条资讯变成知识点，逐条由你语音确认（"入/不要/讲细点"）；一旦入库，**后续的合并/连线/归档全部 AI 自动完成，不再要你确认**。
2. **彻底沉浸**：没有导航栏、收件箱、文档库、审批面板。主画面只有**全屏大脑星图 + 语音光球**，设置缩成角落一个小图标。
3. **自动整理的兜底**（让"自动"安全可后悔）：归档=隐藏不真删；每次结构变更都进**变更历史**且**可一键撤销**；它会**偶尔口头汇报**"我刚把 X 并进了 Y"。

> 决策 1 **刻意推翻**了旧版铁律"任何图谱结构变更都先建议后确认"。这是大脑主人的主动选择，已在 `AGENTS.md` 不变量中同步改写。

## 一、核心体验流程
```
开机
  → 科幻启动场景（电影感开场，非冰冷"系统启动"）
  → 语音自检：逐条念出"麦克风 ✓ / 扬声器 ✓ / 网络 ✓ / 资讯源 ✓ / 大脑读写 ✓"
       （任一项失败 → 语音说明 + 优雅降级，不卡死）
  → 漂亮加载：实时抓当天资讯，"数据流注入大脑"科幻动画，把等待变仪式感
  → 伴侣登场，主动开口（不是等你点按钮）
       │
       ├─ 唠家常 / 开玩笑 / 闲聊 —— 像真人，聊完即丢，不进库
       │
       ├─ 讲资讯（核心）：AI 资讯 + GitHub 趋势（star 增速）
       │    每条：用人话讲原理 → 口播"入库?"
       │      ├ "入"   → 建概念节点（附来源链接）→ ★它自动归纳：自动连到相关概念 /
       │      │           自动融合进已有概念 / 自动归档被取代的旧节点（全程无需你确认）
       │      ├ "不要" → 丢弃，继续下一条
       │      └ "讲细点"→ 再深讲一层，然后再问
       │
       └─ 讲已有知识：指定讲（"讲讲 RAG"）/ 串着讲（"把 Agent 相关梳理一遍"）
            → 顺关系线走一圈，星图节点与连线**随语音同步高亮**
  · 全程你都能随时插话打断，它立刻停下来听（硬需求）
  · 你在闲聊中透露的偏好/已懂/不懂 → 它静默写进"懂你层"，越用越准（无需确认）
```

## 二、三层记忆（产品灵魂，保留）
| 层 | 存什么 | 去留 |
|---|---|---|
| ① 原始音频 / 新闻全文 | 录音、抓来的原文 | **聊完即丢** |
| ② 知识库（大脑星图） | 概念 + 简介 + 关系 + 来源链接 | **永久** |
| ③ 用户画像（懂你层） | 兴趣、已懂/不懂、偏好讲解风格、常问方向、口头习惯 | **永久，持续生长（静默蒸馏，无需确认）** |

丢的是"音频文件"，不是"对你的记忆"。丢弃音频前先把关于你的信息蒸馏进第③层——这就是"越用越懂"。

> **记忆引擎（M 系列，派生层，复用）**：开源 EverMemOS 本机自部署，"先回忆再回答"。它**只读注入对话、只写蒸馏纯文本**，**绝不写图谱/画像**。注意：v2 的"自动整理"是 **agent/curation 层**在写图谱（已获授权），与"记忆引擎不写图谱"是两回事，互不冲突。

## 三、知识库机制（自生长，最核心）
- **入库单位 = 概念 + 简介**，不是新闻碎片；多条同主题新闻喂养同一概念。
- **自动净化更迭（v2 改为自动执行）**：
  - 补充 → AI **自动融合**进已有概念
  - 过时 / 有更优解 → AI **自动归档**旧的（隐藏可恢复，非真删）
  - 关系线随节点更替**自动迁移**，不断裂
- **兜底**：所有自动结构变更进**变更历史**，可**一键撤销**；偶尔语音汇报。
- 来源链接：每个概念附一个链接（只存链接不存全文）。
- 手动编辑节点能力**降级为隐藏入口**（应对偶发讲错），不是主流程。

---

## 四、前端：唯一画面 = 科幻"大脑星图" + 语音光球
- **没有导航栏、没有分区**。整屏就是星图，旁边一颗会随语音律动的**声波光球**。
- 概念为**节点**，支持**可缩放分层**（脑区 → 展开细节）；关系线类型少而精（是一种/依赖/替代/相关）。
- 平时干净：只显线 + 箭头，线型/颜色区分类型，不堆标签。
- **悬停节点**：高亮聚焦、相连线浮现标签 + 弹简短概要；**悬停连线**：显示完整关系描述。
- **串讲同步高亮**：讲到哪，节点和连线亮到哪。
- **设置**：仅角落一个小图标，点开浮层——只有**音色 / 人格预设 / （验收期）API key**，没别的。

---

## 五、语音交互
- 全程语音，体验**像人与人沟通**，不是命令式问答；会唠家常、会开玩笑（吃人格预设）。
- **可打断（硬需求）**：讲到一半你插话，它立刻停下听。这是"像人"的命门。
- 伴侣**主动开口**（v2 改：登场即说话），但讲资讯节奏不啰嗦。
- **音色可控**：你能换它的嗓音（OpenAI Realtime voice 参数；mock 期模拟）。
- 入库快捷口令："入 / 不要 / 讲细点"，每条都问但回答可以很快。

## 六、冷启动（第一天）
定位：**与你共同成长的伴侣**，第一天就"点亮第一颗星"，绝不给你空白。
```
1. 自我介绍（像新朋友）："你好，我还没名字也还不认识你，要不…你先帮我起个名字?"
2. 快速人格 + 音色设置（约 15 秒）：3~4 个"声音+性格"预设，点一下即可。以后可改、会随画像自调。
3. 对话式问 2~3 个兴趣方向（聊天，不是问卷）。
4. ⭐ 当场抓一条当天 AI 新闻，人话讲完问"作为大脑里第一个知识点?" → "要" → 空星图中央亮起第一颗星。
5. "这就是我们一起搭起来的第一块。以后每天它都会长大一点。"
```
情感钩子 = "第一颗星亮起" + "被起名字的伙伴"。

## 八、范围 vs 砍掉的
**v2 必做（V 系列里程碑）**：科幻启动 + 语音自检、开机即抓 + 注入动画、全程可打断语音伴侣（唠嗑/开玩笑/讲资讯/讲已有知识）、语音入库决策（唯一保留的确认）、入库后自动归纳整理（合并/归档/迁移，全自动 + 历史 + 撤销）、画像静默生长、音色 + 人格控制、唯一画面=星图 + 串讲同步高亮。

**明确砍掉（移出主流程）**：左侧导航栏、探索资讯流页、文档库页、思维导图大纲页、提议收件箱 / 审批面板、分析洞察页、完整设置面板（缩成角落图标）、一切需要鼠标逐条点击审批的交互。

**复用底座（不砍）**：`providers/**`（voice/llm/memory/news）、`domain/`、`storage/`、`lib/graphMutations|salience|profileDistillation`、`agent/curation` 的打分/检测逻辑（改为自动执行）、`BrainGraph(3D)View` + 视觉 token、M/H 系列硬化与记忆评测。

**约束（心里有数）**：
- **无法本地部署 ollama**；LLM/语音走云 API（验收期才接真 key）。
- **全程 mock-first**：每个里程碑都要在 mock provider 下端到端可演示，不耗 key；**直到最后验收阶段才接入真实 API key**。
- 成本（STT/TTS/LLM）与中英术语（默认中文讲解、技术词保留英文）。

## 九、存储与隐私
- 知识库 / 画像 = 本地 SQLite 纯文字（1 万知识点 ≈ 10MB）。
- 重计算（语音、大模型）走云 API。
- 本地优先 → 数据不出机器（MVP）。
- 记忆引擎（可选）= 本机自部署 EverMemOS，未起则优雅降级，应用照常可用。
