# my_brain — 产品设计文档
> 这是我们逐条敲定的产品共识，作为开发起点 / 存档。配套 `AGENTS.md` 给 AI 编码助手作技术约束。
## 一句话定义
一个**语音优先、与用户共同成长的 AI 伴侣**：帮用户追 AI 动态（资讯 + GitHub 优质项目），并亲手搭建一个会**自我净化更迭**的"大脑知识库"。用得越久，越懂用户。全程语音交互，像人与人沟通；前端是富有科技感的"大脑星图"。
---
## 一、核心流程
```
开机启动
  → 自检（运行所需的一切就绪：麦克风/扬声器、网络、资讯源、知识库读写等）
  → 漂亮加载（实时抓取当天资讯，配"数据流注入大脑"的科幻动画，把等待变成仪式感）
  → "今天想了解些什么?"
       │
       ├─ AI 资讯（权威渠道 + 小道消息；播报时标注来源，由用户自行判断可信度）
       │    GitHub 项目筛选：star 增速 + 趋势
       │    └ 每条：语音通俗讲解（原理层面、说人话）→ 问"入库?"
       │         └ 入库 → AI 建议挂接到哪个概念 → 用户确认
       │              ├ 全新 → 新建概念节点（附来源链接）
       │              ├ 补充已有 → 融合进已有概念（先建议，用户点头）
       │              └ 旧的过时 / 已不适用 / 有更优解 → 归档旧的（先建议，用户点头）
       │                   └ 旧节点的关系线 → 自动迁移到新节点
       │
       ├─ 其他类目（非 AI）→ 随便聊 → 聊完完全丢弃（不进知识库，除非用户主动提需求）
       │
       └─ 讲已有知识 → 指定讲（"给我讲讲 RAG"）／ 串着讲（"把 AI Agent 相关的梳理一遍"，
                        顺着关系线走一圈，前端节点与连线同步高亮，讲到哪亮到哪）
```

## 二、三层记忆（产品灵魂）
| 层 | 存什么 | 去留 |
|---|---|---|
| ① 原始音频 / 新闻全文 | 录音、抓来的原文 | **聊完即丢**（占空间、无长期价值） |
| ② 知识库（大脑图谱） | 概念 + 简介 + 关系 + 来源链接 | **永久** |
| ③ 用户画像（懂你层） | 兴趣方向、已懂/不懂什么、偏好的讲解风格、常问方向、口头习惯 | **永久，持续生长** |
**关键：丢的是"音频文件"，不是"对用户的记忆"。** 在丢弃音频前，AI 会从对话中提炼关于用户的信息写入第③层 —— 这就是"越用越懂你"的实现方式。三层都是纯文字，几乎不占空间。
---
## 三、知识库机制（最核心、也是壁垒）
- **入库单位 = 概念 + 简介**，不是新闻碎片。多条同主题新闻一起喂养/更新同一个概念节点 → 大脑在"长知识"而非"堆新闻"。
- **净化更迭规则**：
  - 补充 → **融合**进已有概念
  - 过时 / 不再适用 / 有更优解 → **归档旧的**（隐藏但可找回，非真删）
  - 关系线随节点更替**迁移**，不断裂
- **铁律：任何改动图谱结构的操作（融合 / 归档 / 删除 / 挂接 / 连线）都"先建议、用户确认后才执行"。** AI 是帮手，用户是大脑的主人。
- **删除 = 归档**：从图谱视觉和检索中隐藏，可找回，防误伤（纯文字代价几乎为零）。
- 节点**可手动增 / 改 / 删**（应对 AI 通俗讲解可能出现的错误）。
- 每个概念**附一个来源链接**（只存链接不存全文），便于回溯原文。
---
## 四、前端：科幻"大脑星图"
- 概念为**节点**；支持**可缩放分层**（平时看一团团"脑区" → 点进去展开细节，应对节点膨胀）。
- 关系线**类型少而精**（如：是一种 / 依赖 / 替代 / 相关），避免过度复杂。
- 显示策略 —— **信息按需浮现，平时是干净星图**：
  - 静态：只显示**线 + 箭头方向**，用**线型 / 颜色**区分类型，**不堆文字标签**。
  - **悬停节点**：该节点高亮，无关节点/线变暗（聚焦）；直接相连的线浮现文字标签；弹出**简短概要**。
  - **悬停连线**：线加粗，显示完整关系描述（如"Agent 框架 依赖 大模型 API"）。
- 旁边是语音沟通区：实时对话有**声波条**动效，富有科技感。
- 串着讲知识时，节点与关系线**跟随语音同步高亮**。
---
## 五、语音交互
- 全程语音，体验**像人与人沟通**，而非命令式问答。
- **可打断（必须，MVP 即实现）**：助手讲到一半，用户能随时插话，它立刻停下来听。这是"像人"的命门。
- 不主动播报；用户开口才响应（但允许后台静默准备资讯）。
- 入库确认支持快捷口令（"入""不要""讲细点再决定"），每条都问但回答可以很快、不啰嗦。
---
## 六、冷启动（第一天）
定位：**与用户共同成长的伴侣**。绝不让用户面对空白，第一天就要"点亮第一颗星"。
```
1. 自我介绍（像新朋友打招呼，非冰冷"系统启动"）
   "你好，我还没有名字，也还不认识你。要不…你先帮我起个名字?"
2. 快速人格设置（约 15 秒）：给 3~4 个"声音+性格"预设组合
   （如：沉稳导师 / 活泼伙伴 / 冷静极客），点一下即可，绝不久等。
   人格以后可改，并会随用户画像慢慢自我微调。
3. 对话式问 2~3 个兴趣方向（是聊天，不是问卷）。
4. ⭐ 立刻兑现价值：当场抓一条当天 AI 新闻，通俗讲解，
   讲完问"要不要作为你大脑里的第一个知识点?" → 用户说"要" → 空图谱中央亮起第一颗星。
5. "这就是我们一起搭起来的第一块。以后每天，它都会长大一点。"
```
情感钩子 = "第一颗星亮起" + "被起名字的伙伴"。

## 七、形态
桌面程序 + 网页（同一前端，两个目标）。科幻大脑图谱在桌面/大屏体验最佳。
---
## 八、MVP 范围 vs 后续迭代
**MVP 必做**：开机即抓 + 加载动画、全程语音、**可打断**、冷启动流程、知识库三大操作（融合 / 归档 / 关系迁移，均先建议后确认）、分层可缩放大脑图谱、节点手动编辑、来源链接、用户画像层、人格预设。
**后续迭代**：
- **云端组件**（凌晨定时预抓，解决"电脑夜间关机无法预抓"；多设备同步）。MVP 先用"开机即抓 + 漂亮加载"替代。
- 关系类型扩展、数据云备份 / 导出。
**心里有数（非阻塞）**：语音（STT/TTS）+ 大模型调用的云 API 成本；中英术语处理（默认中文讲解，技术术语保留英文原词）。
---
## 九、存储与隐私说明
- 知识库 / 用户画像 = 本地硬盘上的纯文字文件（如 SQLite）。**1 万个知识点 ≈ 10MB**，可忽略。
- 重计算（语音识别、语音合成、大模型）走云端 API，不吃本地内存。
- 本地优先 → 隐私最好，数据不出用户电脑（MVP）。
