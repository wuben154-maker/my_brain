# 默认输出模板（Agent 填实后发给用户，勿原样粘贴占位符）

**规则：**
- 总长度 **≤ 20 行**（合格时 **≤ 8 行**）
- 用中文（用户未要求英文时）
- **合格**：不写 AC 明细、不写 in-scope 文件列表
- **不合格 / 阻塞**：只列失败项，每项一行

---

## 合格（示例形态）

```
AW-01 验收：合格

验证：go test（相关包）✓ · web tsc ✓
范围：无漂移
```

---

## 不合格（示例形态）

```
AW-01 验收：不合格

- AC-02：前端未从事件流渲染错误态（缺测试/实现）
- 验证：`cd web && npm test` exit 1
- 漂移：`web/src/app/console/billing/page.tsx`（超出本 spec 范围）
- 代码：`agent/reducer.ts:42` 未校验 LLM 输出即写入状态（High）
```

---

## 阻塞（示例形态）

```
AW-02 验收：阻塞

- 无法跑 UI 相关 AC：缺登录会话 / MCP 不可用
- 需要：提供测试账号或先 `auth:bootstrap`
```

---

## 字段（Agent 填写）

第一行必须是：`{SPEC_ID} 验收：{合格|不合格|阻塞}`

可选第二行（合格时）：`验证：` + 实际跑过的 1–3 条命令结果  
不合格/阻塞：用 `- ` 列表，**最多 8 条**，按优先级：AC → 验证 → 漂移 → 代码安全问题
